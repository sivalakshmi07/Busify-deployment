\chapter{DEPLOYMENT}

\section{Folder Structure}

The Bus Ticket Booking application follows a structured modular directory architecture. 
The MERN application is divided into two main environments: the React Client (busbooking-app) and the Node Server (MONGG).

\begin{figure}[H]
\centering
\includegraphics[width=0.8\textwidth]{folder_structure.png} 
\caption{Backend and Frontend Folder Structure}
\end{figure}

\textbf{Frontend (busbooking-app)}
\begin{itemize}
    \item \texttt{src/components}: Reusable UI components such as seat matrices, navigation bars, and bus list cards.
    \item \texttt{src/pages}: Route-level components mapping to exact URLs (e.g., Home, SeatSelection, Checkout, Login, AdminDashboard).
    \item \texttt{src/context}: React context providers for persistent user state.
    \item \texttt{package.json}: Handles frontend dependencies (e.g., react-router-dom, axios).
\end{itemize}

\textbf{Backend (MONGG)}
\begin{itemize}
    \item \texttt{models}: Mongoose schema definitions mapping strict properties to the MongoDB documents (User, Bus, Booking).
    \item \texttt{server.js}: Core Express engine handling authentication endpoints, parsing seating updates, and executing memory-based Mutex locking logic.
    \item \texttt{package.json}: Dependencies for the node server (e.g., bcryptjs, jsonwebtoken, mongoose, express).
\end{itemize}
