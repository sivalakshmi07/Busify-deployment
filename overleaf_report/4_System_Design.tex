\section{Architectural Design}

The proposed system is designed using a component-based architecture implemented with React.js, which allows the application to be divided into independent, reusable, and manageable components. Each major functionality of the system—such as user authentication, bus search, seat selection, checkout, invoice generation, and admin management—is developed as a separate component. This approach improves code reusability, maintainability, and scalability.

The architecture follows a modular design, where components interact with each other through well-defined props and state management. For example, the bus search component passes selected route data to the bus results component, which further transfers selected bus details to the seat selection and checkout modules. This clear separation of concerns ensures that changes in one module do not directly affect other modules.

Additionally, routing is handled using React Router, enabling smooth navigation between different pages such as Home, Bus Results, Seat Selection, Checkout, Invoice, and Admin Dashboard without reloading the page. The system also uses browser storage (localStorage) to manage user login state and admin privileges. Overall, the architectural design ensures flexibility, efficient data flow, and ease of future enhancements such as backend integration and database connectivity.

Overall, the system architecture ensures efficiency, security, and a responsive user interface across devices.

\section{Activity Diagram}
